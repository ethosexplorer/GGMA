Section 2

Account & User Onboarding
2.1 Creating a New User Account

There are two ways users join GGP-OS:

ASelf-Registration (Patient/Business)

Users go to www.ggp-os.com → Click "Login/Register" → Create account with email/password or Google. They're automatically assigned a default role based on their registration type.

BAdmin-Created Account (Staff/Internal)

From your Admin Dashboard → User Directory → Click "Create Staff Invite". Enter the new user's email, assign their role, and set their jurisdiction. They'll receive an invitation to create their account.

2.3 Modifying User Roles
Admin Dashboard → User Directory → Hover over user → Click "Edit" → Change role dropdown → Save
⚠️ Role Change Impact: Changing a user's role immediately changes what dashboard they see on their next login. It also updates their data access permissions. Always confirm with the Founder before promoting to admin-level roles.
2.4 Suspending/Deactivating Accounts

If you need to immediately revoke access:

Admin Dashboard → User Directory → Hover over user → Click "Suspend"

This triggers an Emergency Suspension which:

  • Severs all active API keys
  • Locks portal access immediately
  • Logs the action in the immutable audit trail
  • Sends notification to the Founder

Patient Intake Processing — Backend
5.1 Backend Intake Flow

When a patient completes intake, here's what happens behind the scenes and what you need to do:

StepSystem ActionYour Action
1. RegistrationFirebase creates auth account. User doc added to Firestore 'users' collection.None — automatic. Shows up in your Patient Registry.
2. Sylara IntakeAI collects state, condition, preferences. Data stored in user profile.Review intake data for completeness.
3. BookingCalendly event created. Linked to patient record.Ensure provider is available for the time slot.
4. ConsultationProvider marks recommendation in system.Verify recommendation is recorded correctly.
5. PaymentStripe processes payment. Receipt emailed. Revenue logged.Check for failed payments. Follow up on declines.
6. ApplicationPatient submits documents to state portal.Guide patient through state portal if needed. Track submission status.
7. ApprovalState approves/denies. Status updated in system.Notify patient of approval. Update their record status.
5.2 Mississippi-Specific Backend Steps
Mississippi MMCP Process:
  1. Patient sees Mississippi-licensed physician who is registered with MMCP
  2. Physician enters recommendation into MMCP's practitioner portal
  3. Patient goes to medicalcannabis.ms.gov and creates a patient account
  4. Patient uploads: physician recommendation, photo ID, proof of MS residency
  5. Patient pays $25 state application fee online
  6. MSDH reviews and approves (typically 15-30 business days)
  7. Patient receives digital medical cannabis card
  8. Card valid for 1 year from issuance date
5.3 Handling Failed Payments
Admin Dashboard → Check Stripe dashboard (dashboard.stripe.com) → Filter by "Failed" → Contact patient

Common failure reasons:

  • Card declined: Ask patient to try a different card or contact their bank
  • Incorrect billing zip: Patient entered wrong zip code for their card
  • Insufficient funds: Suggest they try again later or use a different payment method
  • 3D Secure failed: Patient needs to complete bank authentication

7.2 Call Handling Best Practices
  • Answer within 3 rings when the Web Dialer shows an incoming call
  • Identify yourself: "Thank you for calling GGP-OS, this is [Name], how can I help you?"
  • Log every call: After hanging up, immediately add notes to the contact's CRM record

Section 10

Security & Compliance Protocols
10.1 HIPAA Compliance

GGP-OS handles protected health information (PHI). You must:

  • Never share patient data outside the platform
  • Never screenshot patient records for personal use
  • Always use secure channels (platform messaging, not personal email)
  • Log out when away from your computer
  • Use strong passwords and enable 2FA if available
🔴 HIPAA Violation Warning: Sharing patient medical information through unsecured channels (personal email, text, social media) is a federal offense. All communication about patient records must happen through the GGP-OS platform or encrypted channels.
10.2 Immutable Audit Trail

Every action in the system is logged and cannot be deleted or modified. The audit trail records:

  • Who made the change (user ID, role, email)
  • What was changed (field, old value, new value)
  • When (timestamp with timezone)
  • Why (action type: manual edit, AI decision, system auto)
